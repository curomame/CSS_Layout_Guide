1. CSS의 역사와 문제점

1.1 CSS의 시작

CSS는 HTML이나 XML의 각 요소를 꾸미는 것이며, 문서의 구조와 스타일을 분리하는 것.
CSS등장 전에는 각 요소의 전용 속성이나 style 속성을 일일이 지정함.
하지만 그 방법은 비효율적이고 그러한 문제를 해결하기 위해 생성된 것이 CCSS임.

1.4 해결책으로 탄생한 CSS 설계

- 규칙을 확실히 정리하지 않으면 혼자 개발을 수행하게 되므로, 코드를 기술하는 방법이 어긋난다.
- 문서화되어 있지 않으면 다른 사람과 협력해서 작업을 수행할 때 코드를 기술하는 방법이 어긋난다.
- 너무 쉽게 생각하기 때문에, 애초에 규칙으로서의 역할을 하지 못한다.

이를 위해 CSS설계라는 것이 생겨남.

다양한 설계방법이 있지만 대표적인 공통점은
1.추상화한다.(기능별로 묶기)
2.나눈다.(파일, 부품 크기, 역할에 따른 이름)

이 책에서는 
OOCSS
SMACSS
BEM
PRECSS
의 방법을 배울 예정

1.5 CSS설계와 디자인 시스템 연동

아토믹 디자인
Atoms => 원자 - 가장 작은 단위가 되는 모듈, 모든 웹 페이지에서 사용하는 것으로 버튼이나 입력 필드, 제목등이 여기에 해당
Molecules => 분자 - 원자가 모여 그룹을 만든 것.
Organisms => 유기체 - 분자뿐 아니라 원자와 다른 유기체를 포함할 수 있음 ex)헤더
Templates - 지금까지 말한 요소들을 조합해 만든 레이아웃
Pages - 실제 이미지나 텍스트 콘텐츠까지 포함한 요소
